## Fn key dictation shortcut

OpenWispr now supports using the Mac `fn` key by itself as the Transcribe
shortcut. If your shortcut settings were still using the older Tauri keyboard
backend, editing the Transcribe shortcut now switches to the HandyKeys backend
first so the `fn` key can be captured and saved correctly.

This update also prevents stale `fn` shortcuts from being registered through
the Tauri backend on startup, so fallback profiles keep a working shortcut
instead of silently failing.
