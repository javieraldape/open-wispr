## README maintenance

Update `README.md` whenever a significant, user-relevant change lands. This
includes product positioning, installation or release flow, supported platforms,
first-run behavior, user-facing workflows, model recommendations, CLI behavior,
troubleshooting steps, licensing/attribution, or any feature that changes what
early users should understand before trying OpenWispr.

Do not update `README.md` for purely internal refactors, tests, formatting, or
implementation details that do not change the user/developer story.
